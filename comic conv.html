Module 0: قصة JavaScript

إيه يا صااااوى! أنا بسمع عن حاجة اسمها JavaScript في كل حتة، إيه قصتها دي؟ هي بتاعة القهوة؟

يا جمعة، قهوة إيه بس! دي لغة برمجة، بس سهلة. زمان، كانت صفحات النت عاملة زي الجرايد، كلام وصور وبس. مملة يعني.

جرايد؟ وبعدين؟

جه واحد فنان اسمه "برندان آيك"، في 10 أيام بس، عمل لغة تخلي الصفحات دي تتكلم وتتحرك. يعني لما تدوس على زرار، حاجة تحصل. فهمت؟

اااه، يعني هي اللي خلت النت فيه أكشن؟ طب ليه اسمها "جافا" سكريبت؟ ليها علاقة بلغة جافا؟

لا خالص، دي حركة تسويق عشان كانت "جافا" مشهورة وقتها. زي ما تسمي ابنك "ميسي" عشان الناس تحبه. مالهمش علاقة ببعض.

<!DOCTYPE html>

Module 1: أنواع البيانات الأساسية

يا صاوى، إيه حوار أنواع البيانات ده؟ أنا كل اللي أعرفه الأرقام والحروف.

بالظبط كده! دي بنسميها Primitive Types. نبدأ بالـ Numbers. في الجافاسكريبت، مفيش فرق بين رقم صحيح ورقم عشري، كلهم واحد.

console.log(3/0);          // Infinity (لا نهاية)
console.log("abc"/2);        // NaN (Not a Number)
console.log(Number.MAX_VALUE); // أكبر رقم ممكن

وخلي بالك، لو حاولت تقارن NaN == NaN هتديك false! استخدم Number.isNaN() عشان تتأكد.

تمام، والنوع التاني هو الـ Strings أو النصوص. دي سهلة.

سهلة بس فيها خدعة. الـ Strings في الجافاسكريبت Immutable، يعني غير قابلة للتغيير. لو حاولت تغير حرف فيها، أنت في الحقيقة بتعمل string جديد خالص.

let phrase = "JavaScript";
console.log(phrase.charAt(4)); // S
console.log(phrase.slice(4,10)); // Script
console.log(phrase.split('a')); // [ 'J', 'v', 'Script' ]

أها! فهمت. طب وإيه موضوع الصح والغلط ده؟ الـ Booleans؟

ممتاز. في الجافاسكريبت، مش بس true و false هما اللي بيعبروا عن الصح والغلط. فيه قيم بنعتبرها "falsy" يعني بتدي معنى الـ false.

القيم الـ Falsy هي: false, 0, "" (نص فاضي), null, undefined, و NaN.

أي حاجة غير دول بنعتبرها Truthy.

if ("hello") { // truthy
  console.log('This will run');
}
if (0) { // falsy
  console.log('This will NOT run');
}

Module 2: Null vs Undefined و Coercion

يا صااااوى، أنا هتجنن! إيه الفرق بين null و undefined؟ الاتنين شكلهم زي بعض، معناهم "فاضي" وخلاص!

اهدأ يا جمعة، الموضوع بسيط. بص، undefined دي بتاعت النظام. معناها "أنا معرفش فيه إيه". زي صندوق بريد جديد لسه محدش حط فيه جوابات. هو موجود، بس حالته غير مُعرَّفة.

let user;
console.log(user); // Output: undefined

آه... يعني النظام هو اللي بيقول معرفش. طب و null؟

null دي بقى بتاعتك أنت كمبرمج. أنت اللي بتروح للصندوق ده وتفضّيه وتقول "أنا عايزه يبقى فاضي عن قصد". أنت اللي بتدي له القيمة دي بنفسك.

let loggedInUser = { name: 'Ahmed' };
// المستخدم عمل تسجيل خروج
loggedInUser = null; // أنت اللي خليته فاضي
console.log(loggedInUser); // Output: null

فهمت! يبقى undefined دي "معرفش"، لكن null دي "فاضي بإيدي". تسلم يا صاوي!

الخلاصة: undefined هي "لسه مفيش قيمة"، أما null هي "القيمة اللي فيه هي 'فارغ'".

صاااوى، الحقني! بص الكود ده، ليه لما بسأل عن نوع null بيقولي 'object'؟ مش المفروض يبقى 'null'؟

typeof null; // 'object' (WTF?!)

ههههه، دي غلطة قديمة في اللغة نفسها وفضلت موجودة عشان مايكسروش الكود القديم. اعتبرها علامة مميزة كده، الكل عارفها. فكك منها.

ماشي هعديها... طب بص المصيبة دي! ليه '5' + 3 بتطلع '53'، لكن '5' - 3 بتطلع 2؟ ده تهريج!

'5' + 3; // '53'
'5' - 3; // 2

ده مش تهريج، ده اسمه Coercion. بص، علامة الزائد + ليها شخصيتين: جامعة أرقام، و لازقة كلام (strings).

لو شافت string في أي حتة، بتتحول للازقة على طول وتلزق كله في بعض. دي اسمها Concatenation.

'5' + 3 // '53' (الـ + شاف الـ string فقال: أنا لازقة)

لكن باقي العمليات (-, *, /) ملهاش غير شخصية واحدة: بتاعت حساب. فبتحاول تجبر أي حاجة مش رقم إنها تبقى رقم عشان تحسبها. ده اسمه Arithmetic Coercion.

'5' - 3 // 2 (الـ - معندهاش غير شخصية واحدة: الحساب)

آهااا! يعني الزائد هو بس اللي بيستهبل، الباقي محترمين! وصلت يا كبير!

الخلاصة: الـ + حساس للـ strings، باقي العوامل الحسابية حساسة للـ numbers.

جرب بنفسك: Number(""), Boolean("0").

Module 3: ليه الصاوى زعلان من var؟

صاوى، أنا كتبت كود أهو، بس مش فاهم ليه لما بطلب سنّي قبل ما أقوله، بيقولي undefined ومش بيزعقلي؟

console.log(age); // طلع لي: undefined
var age = 30;

عشان دي حركة var الصايعة اللي اسمها Hoisting. بتخلي جافاسكريبت تاخد إعلان المتغير بتاعك ترفعه فوق خالص قبل ما تشغل الكود، وتديله قيمة undefined مؤقتًا. كأنك بتقول "فيه متغير اسمه age جاي في السكة".

طب والمصيبة التانية دي؟ إزاي أقدر أعمل متغير بنفس الاسم مرتين؟ ده لخبطة!

var myVar = 'الأول';
var myVar = 'التاني'; // عادي؟!
console.log(myVar); // طلع لي: التاني

بالظبط! عشان كده بنستخدم let و const. دول أولاد ناس ومبيسمحوش بالعبط ده.

مع let، لو حاولت تستخدم المتغير قبل ما تعلنه، هتاخد Error في وشك على طول. مفيش هزار. المنطقة دي اسمها Temporal Dead Zone، يعني منطقة الموت المؤقتة، ممنوع اللمس.

console.log(name); // ERROR! Cannot access 'name'
let name = 'Sawy';

وكمان بيمنعوا إنك تعرف نفس المتغير مرتين. أمان ليك وللكود بتاعك.

let color = 'blue';
let color = 'red'; // ERROR! 'color' has already been declared

الخلاصة: خليك في المضمون واستخدم let و const، وسيبك من var ومشاكله.

Module 4: الصاوى وجمعة في جولة سريعة على الـ Operators

يا صاوى، عايز أعرف الـ Operators دي بتعمل إيه بسرعة. نبدأ بالـ Arithmetic؟

دي بتاعة الحسابات الأساسية: جمع، طرح، ضرب، قسمة، وباقي القسمة (%).

console.log(10 + 3); // 13
console.log(10 % 3); // 1 (عشان 10/3 فيها 3 والباقي 1)

طب والـ ++ والـ --؟

دي بتزود أو تنقص واحد. بس خد بالك، مكانها بيفرق. قبل المتغير (++n) يزوّد الأول، بعده (n++) يستخدم القيمة القديمة الأول بعدين يزوّد.

let n = 5;
console.log(++n); // 6
console.log(n++); // 6 برضه (بس n بقت 7)
console.log(n);   // 7

والـ Assignment اللي هي += و *=؟

دي اختصارات. بدل ما تقول x = x + 5، بتقول x += 5. أسهل وأسرع.

let x = 10;
x -= 3; // x بقت 7
x *= 2; // x بقت 14

أهم سؤال: إيه الفرق بين == و ===؟

=== (3 يساووا) صارمة وبتقارن القيمة والنوع. == (2 يساووا) متساهلة وبتحاول تغير النوع قبل ما تقارن، وده بيعمل مشاكل. خليك دايماً في ===.

console.log(5 == '5');  // true (متساهلة)
console.log(5 === '5'); // false (صارمة)

والـ Logical (&&, ||)؟

دي بتعمل حاجة اسمها Short-Circuit. || بتدور على أول قيمة حقيقية (truthy) وتقف. && بتدور على أول قيمة مزيفة (falsy) وتقف. مفيدة جداً في تحديد قيم افتراضية.

let username = null || 'Guest'; // username = 'Guest'
let data = 'Data' && 0 && 'More'; // data = 0

آخر حاجة، الـ Ternary Operator اللي شكلها غريب دي؟

دي اختصار لجملة if/else. صيغتها: (الشرط) ? (لو صح) : (لو غلط).

let age = 20;
let message = age >= 18 ? 'أهلاً بيك' : 'ممنوع الدخول';
// message = 'أهلاً بيك'

وبكده يا جمعة لمينا معظم الـ Operators المهمة!

Module 5: الفاصلة المنقوطة الشبح (ASI)

يا صاوى الحقني! الكود ده المفروض يرجع object، ليه بيرجع undefined؟ أنا هتجنن!

function giveMeObject() {
  return
  { key: 'value' }
}

giveMeObject(); // undefined ؟!

أهلاً بيك في عالم الـ Automatic Semicolon Insertion (ASI). جافاسكريبت شافت كلمة return لوحدها في سطر، راحت حاطة بعدها فاصلة منقوطة (;) من نفسها عشان تخدمك. بكده، الـ function بتاعتك عملت return; والـ object اللي تحت ده بقى كود ملوش لازمة.

يا نهار أبيض! يعني هي بتحط فواصل من عندها؟ طب والحل إيه؟

الحل بسيط، متفصلش بين return والقيمة اللي بترجعها. خلي القوس أو القيمة في نفس السطر. القاعدة الذهبية: Never put a newline after `return`.

function giveMeObject() {
  return { // خليها جنبها كده
    key: 'value'
  }
}

giveMeObject(); // { key: 'value' }

ده بيخلي نيتك واضحة للكود وبيمنع الـ ASI من التدخل بشكل غلط.

Module 6: مصنع الأكواد (Functions)

يا صاوى، أنا بكتب كود كتير بيتكرر. عايز أعمل حاجة زي "مصنع" صغير، أديله شوية حاجات ويطلعلي نتيجة. إيه الحل؟

الحل هو الـ Functions يا جمعة. دي عبارة عن بلوكات كود بتديها اسم، وتقدر تنادي عليها في أي وقت. فيه طريقتين مشهورين عشان تعملها.

1. Function Declaration (الإعلان الصريح):

// بنعلن عن مصنع اسمه add
function add(x, y) {
  return x + y;
}

add(5, 3); // 8

الميزة هنا إنها بيحصلها Hoisting، يعني تقدر تنادي عليها حتى لو كتبتها قبل ما تعرفها.

2. Function Expression (التعبير الوظيفي):

// بنخزن المصنع في متغير
const multiply = function(x, y) {
  return x * y;
};

multiply(5, 3); // 15

دي مبتعملش Hoisting، وده بيخلي الكود منظم أكتر. أغلب الناس بتفضل الطريقة دي مع const.

حلو أوي. طب لو عايز أعمل function تجمع أي عدد من الأرقام أديهولها؟ مش لازم أحدد x و y بس؟

سؤال ممتاز! هنا بنستخدم حاجة اسمها arguments. ده object شبه الـ array، بيلملك كل الحاجات اللي بعتها للـ function.

function sumEverything() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}

sumEverything(1, 2, 3, 4); // 10

ملحوظة: في الجافاسكريبت الحديثة (ES6+)، فيه طريقة أحسن اسمها Rest Parameters، بس خليك عارف دي عشان لو شفتها في كود قديم.

طب افرض عملت متغير جوه الـ function ونسيت أكتب let أو const. إيه اللي هيحصل؟

دي كارثة يا جمعة! لو عملت كده، المتغير ده بيتسرب بره الـ function وبيبقى Global Variable، يعني أي حد في أي حتة في الكود يقدر يشوفه ويغيره ويبوظلك الدنيا.

function createLeak() {
  leakedVar = 'أنا تسربت!';
}

createLeak();
console.log(leakedVar); // "أنا تسربت!"

الحل؟ حط "use strict"; في أول ملف الجافاسكريبت بتاعك. ده بيشغل "الوضع الصارم" اللي بيمنع الأخطاء الشائعة دي وبيطلعلك error واضح.

فهمت! طب ما تديني تمرين أحلّه عشان أثبت المعلومة.

تمرين على مزاجك. اكتبلي function اسمها reverseString تاخد string وتعكسه. يعني لو اديتها 'hello' ترجع 'olleh'.

أعتقد دي طريقة كويسة؟

function reverseString(str) {
  let reversed = '';
  for (let i = str.length - 1; i >= 0; i--) {
    reversed += str[i];
  }
  return reversed;
}

reverseString('Sawy'); // 'ywaS'

الله ينور عليك يا جمعة! بالظبط كده. أنت بقيت برنس في الـ functions.

Module 7: خلي الكود يختار (Control Flow)

يا صاوى، عايز الكود بتاعي يبقى ذكي. يعني لو درجة الطالب فوق 90 ياخد "ممتاز"، لو أقل يبقى "جيد جداً" وهكذا. إزاي أخليه ياخد قرارات؟

دي سهلة، بنستخدم if / else if / else. دي بتخليك تحط شروط، والكود يمشي على أول شرط يتحقق.

let score = 85;

if (score >= 90) {
  console.log('ممتاز');
} else if (score >= 80) {
  console.log('جيد جداً'); // الشرط ده هيتحقق
} else {
  console.log('مقبول');
}
// Output: جيد جداً

تمام! طب لو عايز أطبع الأرقام من 1 لـ 5؟ مش معقول هكتب console.log خمس مرات!

أكيد لأ. هنا يجي دور الـ Loops (الحلقات التكرارية). لو عارف عدد المرات بالظبط، استخدم for loop.

//  (البداية; الشرط; الخطوة)
for (let i = 1; i <= 5; i++) {
  console.log(i);
}
// 1, 2, 3, 4, 5

طب لو عايز أفضل أكرر حاجة طول ما شرط معين متحقق، بس مش عارف كام مرة بالظبط؟

في الحالة دي، استخدم while loop. بتفضل شغالة طول ما الشرط بتاعها true.

let num = 1;
while (num < 50) {
  num = num * 2;
  console.log(num); // 2, 4, 8, 16, 32, 64
}
// بتقف لما الرقم يبقى أكبر من 50

فهمت! طب إديني تحدي يجمع بين الـ loops والـ if.

تحدي جميل. عندك array أرقام، وعايزك تعمل function تلاقي أول رقم زوجي وترجعه. لو مفيش، ترجع رسالة تقول 'No even numbers found'.

let numbers = [1, 3, 5, 8, 9, 10];

أعتقد حليتها!

function findFirstEven(arr) {
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] % 2 === 0) {
      return arr[i]; // لقيت واحد، ارجع بيه فوراً
    }
  }
  return 'No even numbers found'; // خلصت اللفة وملقتش
}

findFirstEven(numbers); // 8

عاش يا وحش! استخدامك للـ return جوه الـ loop عشان توقفها أول ما تلاقي الحل ممتاز. أنت كده فهمت التحكم في المسار صح.

Module 8: لمّة العيلة (Arrays)

يا صاوى، عندي مشكلة. عايز أخزن قايمة بأسماء أصحابي، وقايمة تانية بدرجاتهم في لعبة. عامل متغير لكل واحد والدنيا باظت مني.

أنت محتاج حاجة تلملك الداتا دي كلها في مكان واحد. أنت محتاج Array. فكر فيها كأنها "عيلة" أو "قايمة" بتشيل كل القيم بتاعتك بالترتيب.

// بنعمل array فاضية
let friends = [];

// نضيف عناصر بالترتيب
friends.push('Ahmed');
friends.push('Sara');
friends.push('Mona');

console.log(friends); // ['Ahmed', 'Sara', 'Mona']
console.log(friends[0]); // 'Ahmed' (بنجيب أول عنصر)

حلو أوي! طب لو عندي array فيها أرقام، وعايز أجيب مجموعهم وأعلى درجة؟

let scores = [88, 92, 75, 99, 80];

هنا بقى بنستخدم الـ loops اللي اتعلمناها عشان نلف على الـ array عنصر عنصر.

let scores = [88, 92, 75, 99, 80];
let sum = 0;
let maxScore = scores[0]; // بنفترض إن أول عنصر هو الأعلى مؤقتاً

for (let i = 0; i < scores.length; i++) {
  sum += scores[i]; // بنضيف كل درجة للمجموع

  if (scores[i] > maxScore) {
    maxScore = scores[i]; // لو لقينا درجة أعلى، بنغير الـ max
  }
}

console.log('Sum:', sum); // 434
console.log('Max Score:', maxScore); // 99

بنفس الطريقة تقدر تجيب أقل قيمة أو المتوسط الحسابي.

فكرة جامدة. طب لو عايز أعمل array جديدة تكون فيها كل الدرجات مضروبة في 2؟

سهلة. بتعمل array جديدة فاضية، وتلف على القديمة، وكل مرة تضرب الرقم في 2 وتعمله push في الجديدة.

let scores = [10, 20, 30];
let doubledScores = [];

for (let i = 0; i < scores.length; i++) {
  doubledScores.push(scores[i] * 2);
}

console.log(doubledScores); // [20, 40, 60]

في المستقبل هتتعلم طرق مختصرة زي .map()، بس فهم الأساس ده أهم حاجة.

Module 9: اللعب في الصفحة (DOM Interaction)

يا صاوى، أنا زهقت من الكلام النظري. مش ممكن نلعب في الصفحة دي نفسها؟ يعني أدوس على زرار يغير الكلام ده مثلاً؟

أكيد طبعاً! ده بقى سحر الـ DOM أو (Document Object Model). المتصفح بيشوف صفحة الـ HTML بتاعتك كشجرة من العناصر، وإحنا بالجافاسكريبت نقدر نمسك أي "فرع" في الشجرة دي ونغيره.

بص كده، عندنا برجراف واخد id="welcome-msg". شوف إزاي هنغير محتواه:

// HTML: <p id="welcome-msg">Hello World</p>

// 1. بنمسك العنصر بالـ ID بتاعه
const messageElement = document.getElementById('welcome-msg');

// 2. بنغير النص اللي جواه
messageElement.textContent = 'أهلاً بيك في عالم الـ DOM!';

الله! طب لو عايز آخد كلام من المستخدم نفسه وأعرضه بحروف كبيرة؟

ممتاز. هنعمل input للمستخدم وزرار. لما يدوس على الزرار، ناخد القيمة من الـ input ونعرضها.

// HTML:
// <input id="userInput">
// <button id="actionBtn">اعرض</button>
// <p id="display"></p>

const input = document.getElementById('userInput');
const btn = document.getElementById('actionBtn');
const display = document.getElementById('display');

btn.onclick = function() {
  // بنجيب القيمة من مربع النص
  const userText = input.value;
  // بنعرضها في البرجراف بحروف كبيرة
  display.textContent = userText.toUpperCase();
};

جااامد! طب وصورتك دي، ينفع أكبرّها كل ما أدوس عليها؟

ينفع ونص. هنمسك الصورة ونديها onclick هي كمان، وكل مرة نزوّد العرض بتاعها.

// HTML: <img id="sawy-pic" src="..." width="100">

const sawyPic = document.getElementById('sawy-pic');

Module 10: المشروع النهائي (Mini Lab)

إيه يا جمعة، مستعد للتحدي الأخير؟ هنبني برنامج صغير يجمع كل اللي اتعلمناه. هنعمل "محلل أرقام".

المطلوب: المستخدم يدخل أرقام، والبرنامج يطلعله قايمة الأرقام، عددهم، مجموعهم، والمتوسط بتاعهم. وطبعاً لازم نمنعه يدخل أي حاجة غير الأرقام.

يا دين النبي! أجمع كله في مكان واحد؟ طب إزاي أمنعه يدخل حروف؟

بالـ Validation. هنعمل function صغيرة تتأكد من المدخلات. لو مش رقم، ترجع رسالة خطأ.

function validateInput(text) {
  if (text.trim() === '') {
    return { valid: false, message: 'مينفعش تسيبها فاضية!' };
  }
  const num = Number(text);
  if (isNaN(num)) {
    return { valid: false, message: 'من فضلك أدخل رقم صحيح.' };
  }
  return { valid: true, value: num };
}

فهمتك. طبقت الفكرة وعملت الـ HTML والـ JavaScript. شوف كده وقولي رأيك.

<!-- HTML -->
<h3>Number Analyzer</h3>
<input id="numInput" placeholder="Enter a number">
<button id="addBtn">Add Number</button>
<p id="errorMsg" style="color:red;"></p>
<hr>
<h4>Results:</h4>
<p><b>Numbers:</b> <span id="numList">[]</span></p>
<p><b>Count:</b> <span id="count">0</span></p>
<p><b>Sum:</b> <span id="sum">0</span></p>
<p><b>Average:</b> <span id="avg">0</span></p>
// JavaScript
const numbers = [];

// ... (validation function here)

document.getElementById('addBtn').onclick = function() {
  const inputEl = document.getElementById('numInput');
  const errorEl = document.getElementById('errorMsg');
  const result = validateInput(inputEl.value);

  if (!result.valid) {
    errorEl.textContent = result.message;
    return;
  }

  errorEl.textContent = '';
  numbers.push(result.value);
  inputEl.value = '';

  updateResults();
};

function updateResults() {
  let sum = 0;
  for(let i = 0; i < numbers.length; i++) {
    sum += numbers[i];
  }
  const avg = numbers.length > 0 ? sum / numbers.length : 0;

  document.getElementById('numList').textContent = JSON.stringify(numbers);
  document.getElementById('count').textContent = numbers.length;
  document.getElementById('sum').textContent = sum;
  document.getElementById('avg').textContent = avg.toFixed(2);
}

Module 11: بناء المصنع (Adding Elements to DOM)

يا صاوى، أنا دلوقتي بعرف أغير الحاجات الموجودة. إزاي بقى أبني عناصر جديدة خالص من الصفر وأحطها في الصفحة؟

سؤال المبرمجين الكبار ده. عندك كذا طريقة، كل واحدة ليها استخدامها.

الطريقة الأولى: innerHTML (السهل الممتنع)

دي أسهل طريقة، بتكتب الـ HTML كأنه نص. بس خلي بالك، بتمسح كل حاجة جوه العنصر الأب وتحط الجديد، وممكن تكون خطر لو بتعرض بيانات من المستخدم من غير فلترة.

const container = document.getElementById('container');
container.innerHTML = '<h3>Title</h3><p>New paragraph.</p>';

الطريقة الثانية: createElement (الطريقة الاحترافية)

دي الطريقة الصح والأكتر أمانًا. بتبني كل عنصر لوحده كأنه قطعة ليجو، وبعدين تركبهم في بعض وفي الصفحة.

// 1. Create the new element
const newP = document.createElement('p');

// 2. Set its content and attributes
newP.textContent = 'This is a safe paragraph.';
newP.className = 'highlight';

// 3. Add it to the page
const container = document.getElementById('container');
container.appendChild(newP);

حلوة أوي طريقة الليجو دي. طب إيه حوار document.write اللي بسمع عنه؟

تحذير: document.write()

ابعد عنها! دي طريقة قديمة جداً. لو استخدمتها بعد ما الصفحة تكون حملت، هتمسح كل حاجة في الصفحة وتكتب اللي أنت عايزه بس. محدش بيستخدمها تقريبًا في الشغل الحديث.

// Just for demonstration, DON'T USE THIS!
document.write('<h1>I wiped the page!</h1>');

خليك في createElement عشان تبقى برنس.

Module 12: تطويرات العصر (ES6+ Features)

يا صاوى، بسمع عن حاجات شكلها غريب في الجافاسكريبت الجديدة. سهم كده => وكلمة const. إيه الكلام ده؟

دي يا سيدي تطويرات العصر، اسمها ES6 وما بعدها. دي خلت شكل الكود أحسن وأقوى. نبدأ بالسهل، الـ Arrow Functions.

بدل ما تكتب function() {...}، بتكتبها كده:

// زمان
function add(a, b) {
  return a + b;
}

// دلوقتي
const add = (a, b) => a + b;

مجرد شكل مختصر وأنيق.

حلوة دي! طب و let و const؟ إيه فرقهم عن var اللي صدعتني بيها؟

فاكر لما var كانت بتعمل hoisting وتطلع فوق الـ scope؟ الاتنين دول حلوا المشكلة دي. ليهم Block Scope، يعني بيعيشوا بس جوه القوسين {...} اللي اتعرفوا فيهم.

  • let: للمتغيرات اللي قيمتها ممكن تتغير.
  • const: للثوابت اللي قيمتها مش هتتغير أبدًا.

نصيحة: استخدم const دايمًا، إلا لو كنت متأكد إنك هتحتاج تغير القيمة، ساعتها استخدم let. ارمي var في الزبالة.

أوكي، يبقى const هو الأساس. طب لو عايز أدمج متغير مع نص، كنت بعمل 'Hello ' + name. فيه طريقة أشيك؟

أكيد! استخدم الـ Template Literals. بتستخدم علامة (`) بدل (') أو (")، وتقدر تحط المتغيرات جوه النص علطول.

const user = 'Gomaa';
const greeting = `Hello, ${user}! Welcome back.`;

console.log(greeting); // Hello, Gomaa! Welcome back.

يا سلام! دي هتوفر عليا كتير. إيه تاني من كنوز ES6 دي؟

عندك الـ Destructuring، دي بتخليك تفكك الـ objects والـ arrays بسهولة.

// بتفكك الأوبجكت لمتغيرات
const person = { name: 'Sawy', age: 35 };
const { name, age } = person;
console.log(name); // 'Sawy'

// بتفكك الأراي بالترتيب
const colors = ['Red', 'Green'];
const [firstColor, secondColor] = colors;
console.log(firstColor); // 'Red'

وعندك الـ Rest/Spread Operator (...) اللي بيلم أو يفرد عناصر. دي مواضيع متقدمة شوية بس اعرفها بالاسم دلوقتي.
