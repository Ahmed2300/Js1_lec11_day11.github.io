data_objectModule 1: The Simplest Form - Object Literals

الـ Object هو حجر الأساس في جافاسكريبت. إنه ببساطة حاوية غير مرتبة من البيانات (properties) والسلوكيات (methods) المرتبطة ببعضها. أسهل طريقة لإنشاء object هي باستخدام الـ "Object Literal Syntax" {...}.

Analysis of Object.js

في ملف الديمو، نرى تعريف BookObject. هذا المثال يوضح لنا قوة الـ Literals:

  • Properties متنوعة: يمكن للـ property أن تكون أي نوع بيانات: Name: "JS" (string), version: 2 (number), publishedCities: ["Cairo", "Alex"] (array), وحتى object آخر متداخل مثل "Auther Name".
  • Methods: يمكن أن تكون الـ property عبارة عن function، وفي هذه الحالة نسميها method. مثل Borrow: function(...).
  • الوصول للبيانات: نستخدم إما Dot Notation (BookObject.Name) أو Bracket Notation (BookObject["Auther Name"]). نلجأ للـ Bracket Notation عندما يحتوي اسم الـ property على مسافات أو رموز خاصة، أو عندما يكون اسم الـ property ديناميكيًا (متغير).
const BookObject = {
    Name: "JS",
    "Auther Name": { FName: "John" },
    Borrow: function (readerName) {
        // 'this.Name' is better than 'BookObject.Name' here
        console.log(readerName + " borrowed the book: " + this.Name);
    }
};

// Accessing properties
console.log(BookObject.Name); // "JS"
console.log(BookObject["Auther Name"].FName); // "John"

// Calling a method
BookObject.Borrow("Ahmed");

Dynamic Nature of Objects

جافاسكريبت لغة ديناميكية، مما يعني أنه يمكنك إضافة أو حذف خصائص من الـ object في أي وقت بعد إنشائه.

// Add a new property at runtime
BookObject.pages = 500;
console.log(BookObject.pages); // 500

// Delete a property
delete BookObject.version;
console.log(BookObject.version); // undefined

engineeringModule 2: Creating Objects - Factory vs. Constructor

عندما نحتاج إلى إنشاء العديد من الـ objects التي لها نفس الهيكل، يصبح تكرار الـ object literal أمرًا غير عملي. هنا تظهر الأنماط (Patterns) لإنشاء الكائنات.

Factory Functions (الدوال المصنعية)

هي مجرد دالة عادية تقوم بإنشاء object وإعادته. هي طريقة بسيطة ومباشرة.

Analysis of factoryFunction.js

function StudentFactory(_name, id, age) {
    // 1. Create an empty object
    const studentObject = {};
    
    // 2. Add properties to it
    studentObject.Name = _name;
    studentObject.Id = id;
    studentObject.Age = age;
    
    // 3. Return the new object
    return studentObject;
}

const std1 = StudentFactory("ahmed ali", 1002, 23);
console.log(std1.Name); // "ahmed ali"

Constructor Functions (الدوال البَنَّاءة)

هذه هي الطريقة التقليدية والأكثر شيوعًا لمحاكاة الـ "Classes" في جافاسكريبت (قبل ES6). تعتمد على كلمتين رئيسيتين: this و new.

Analysis of ConstructorFunction.js

عندما نستدعي دالة باستخدام كلمة new، تحدث أربع خطوات سحرية:

  1. يتم إنشاء object فارغ جديد {}.
  2. كلمة this داخل الدالة تشير إلى هذا الـ object الجديد.
  3. يتم تنفيذ الكود داخل الدالة، مما يضيف خصائص للـ object الجديد عبر this.
  4. يتم إرجاع الـ object الجديد تلقائيًا (بشكل ضمني).
// Conventionally, constructor names start with a capital letter
function Student(_Name, _Age) {
    // 'this' refers to the new empty object being created
    this.Name = _Name;
    this.Age = _Age;

    this.toString = function() {
        return "Student Name: " + this.Name;
    }
}

// The 'new' keyword is crucial!
const student1 = new Student("Ahmed Ali", 25);
console.log(student1.Name); // "Ahmed Ali"
console.log(student1.toString()); // "Student Name: Ahmed Ali"

Constructor vs. Factory: ما هو الأفضل؟

الـ Constructor Functions هي المعيار الكلاسيكي وتسمح بالميراث (inheritance) من خلال الـ prototype، وهو مفهوم متقدم. الـ Factory Functions أبسط ولا تستخدم new أو this بنفس الطريقة، مما يراه البعض أسهل وأقل عرضة للأخطاء. في الكود الحديث (ES6+)، نستخدم الكلمة class، وهي في الحقيقة مجرد "sugar coating" فوق الـ Constructor Functions.

privacy_tipModule 3: Encapsulation - Private Properties

أحد المبادئ الأساسية في البرمجة الشيئية (OOP) هو Encapsulation، والذي يعني إخفاء تفاصيل التنفيذ الداخلية للكائن وعدم السماح بالوصول المباشر إليها. في جافاسكريبت، يمكننا محاكاة الخصائص "الخاصة" (private) باستخدام المتغيرات المحلية داخل الـ constructor.

Analysis of Salary in ConstructorFunction.js & Crud.js

لاحظ في الديمو أن salary لم يتم تعريفها كـ this.salary. تم تعريفها كمتغير محلي (local variable) داخل الـ constructor. هذا يعني أنه لا يمكن الوصول إليه من خارج الـ object.

الطريقة الوحيدة للتعامل معه هي من خلال الـ "privileged methods" setSalary و getSalary، لأن هذه الدوال تم تعريفها في نفس النطاق (scope) ولديها إمكانية الوصول إلى المتغير salary.

function Student(_Name, _Age) {
    this.Name = _Name;
    this.Age = _Age;
    
    // This is a "private" variable. It only exists inside this function.
    let salary; 

    // This is a "privileged" method because it has access to the private scope.
    this.setSalary = function(value) {
        if (value > 0) { // We can add validation logic here!
            salary = value;
        }
    }

    this.getSalary = function() {
        return salary;
    }
}

const student1 = new Student("Nadia", 23);
student1.setSalary(5000);

console.log(student1.getSalary()); // 5000
console.log(student1.salary);      // undefined -> Cannot access it directly!

appsModule 4: Putting It All Together - The CRUD Application

الآن سنقوم بتحليل تطبيق الـ CRUD (Create, Read, Update, Delete) من الديمو. هذا المثال يربط كل ما تعلمناه: الـ Constructor Functions، المصفوفات، التلاعب بالـ DOM، والتعامل مع الأحداث (Events).

Analysis of Crud.js

  1. Data Storage: var StudentArray = [];
    هذه هي "قاعدة البيانات" البسيطة الخاصة بنا. إنها مجرد مصفوفة عامة ستخزن كل كائنات الطلاب التي ننشئها.
  2. Create (C): AddnewStudent()
    هذه الدالة هي نقطة البداية. عند النقر على الزر، تقوم بـ:
    • قراءة البيانات من حقول الإدخال (getInputsData).
    • إنشاء كائن طالب جديد باستخدام new Students(...).
    • إضافة الكائن الجديد إلى StudentArray.
    • استدعاء دالة لرسم صف جديد في جدول الـ HTML (createNewTR).
  3. Read (R): createNewTR() & Table Display
    عملية "القراءة" هنا هي عرض البيانات في الجدول. الدالة createNewTR تأخذ كائن طالب وتحوله إلى صف <tr> مع خلايا <td>، ثم تضيفه إلى الـ DOM.
  4. Update (U): FindIndexByID()
    على الرغم من أنها ليست متصلة بزر في الديمو، إلا أن هذه الدالة توضح منطق التحديث. تبحث في المصفوفة عن كائن معين (باستخدام .find()) ثم تعدل إحدى خصائصه مباشرة.
  5. Delete (D): (Not fully implemented in demo, but logic would be similar)
    لإضافة الحذف، سنقوم بإنشاء زر "Delete" في كل صف. عند النقر عليه، سيقوم باستدعاء دالة تعرف index الصف، تستخدم StudentArray.splice(index, 1) لحذف الكائن من المصفوفة، ثم تعيد رسم الجدول بأكمله.

النمط الأهم: Data is the Source of Truth

لاحظ أننا لا نقوم بتعديل جدول الـ HTML مباشرة. بدلاً من ذلك، نحن دائمًا نعدل على مصفوفة البيانات StudentArray أولاً، ثم نعيد رسم الجدول من البداية بناءً على البيانات المحدثة. هذا هو المبدأ الأساسي الذي تعمل به جميع مكتبات الواجهة الأمامية الحديثة مثل React و Vue. البيانات هي مصدر الحقيقة الوحيد، والواجهة هي مجرد انعكاس لها.
